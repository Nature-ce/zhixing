# AI 拆解通过自建后端代理调用

Android 客户端不直接调用大模型 API，而是请求自建后端服务，由后端持有 API Key 并转发给闭源大模型。

## Considered Options

- **端侧直调**：app 直接持有 API Key 请求大模型。被拒：API Key 随客户端分发，反编译可提取，一旦泄露成本风险失控；且 prompt 调优需要发版。
- **端侧 on-device 模型**：完全在手机上跑开源小模型。被拒：当前端侧模型对"任务拆解"这类推理任务质量不稳定，不值得牺牲拆解质量。
- **自建后端代理**（选中）：后端持有 Key、控制配额、调优 prompt 不发版。引入后端基建成本，但在 MVP 阶段优先保障安全与可迭代性。
